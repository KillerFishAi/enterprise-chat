# 把代码推送到 GitHub / Gitee

本地已经完成：`git init`、`git add`、`git commit`，只差「添加远程仓库并推送」。

---

## 第一步：在网站上新建一个空仓库

### 如果用 GitHub
1. 打开 https://github.com 并登录
2. 右上角点击 **"+"** → **"New repository"**
3. **Repository name** 随便填，例如：`enterprise-chat`
4. 选 **Private**（私有）或 Public（公开）
5. **不要**勾选 "Add a README file"
6. 点 **"Create repository"**
7. 创建好后，页面上会有一个地址，复制它，例如：
   - HTTPS：`https://github.com/你的用户名/enterprise-chat.git`
   - 或 SSH：`git@github.com:你的用户名/enterprise-chat.git`

### 如果用 Gitee（国内）
1. 打开 https://gitee.com 并登录
2. 右上角 **"+"** → **"新建仓库"**
3. **仓库名称**随便填，例如：`enterprise-chat`
4. **不要**勾选 "使用 Readme 文件初始化"
5. 点 **"创建"**
6. 创建好后复制仓库地址，例如：
   - HTTPS：`https://gitee.com/你的用户名/enterprise-chat.git`

---

## 第二步：在本地添加远程仓库并推送

在 **PowerShell** 里进入项目根目录，执行（把下面的地址换成你刚复制的）：

```powershell
cd "c:\Users\Administrator\Desktop\Enterprise-chat-layout-main\Enterprise-chat-layout-main"

# 添加远程仓库（替换成你的仓库地址）
git remote add origin https://github.com/你的用户名/enterprise-chat.git

# 推送到远程（第一次推送）
git push -u origin master
```

如果 GitHub 提示你分支叫 `main` 而不是 `master`，可以改成：

```powershell
git branch -M main
git push -u origin main
```

---

## 第三步：输入账号密码（或 Token）

- **GitHub**：现在一般不让用密码，要用 **Personal Access Token**。  
  在 GitHub → Settings → Developer settings → Personal access tokens 里生成一个，推送时「密码」那里粘贴这个 Token。
- **Gitee**：可以用账号 + 密码，或设置 SSH 密钥后用 SSH 地址推送。

---

## 说明

- 项目里的 **`.env`** 没有也不会被提交（在 .gitignore 里），这样更安全。  
  部署到服务器时，在服务器上按 `.env.example` 自己建一个 `.env` 即可。
- 以后改完代码想再上传，只需执行：
  ```powershell
  git add .
  git commit -m "说明你改了什么"
  git push
  ```
